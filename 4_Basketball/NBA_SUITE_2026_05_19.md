## Methodology stack — what makes this rigorous

### 1. Pre-registration discipline (load-bearing)

Every model class ships a pre-registered hypothesis document BEFORE fitting:

- [PRE_REG_SCOUTS_v1.md](D:/Draft%20Resolve/docs/PRE_REG_SCOUTS_v1.md) — 4 falsification triggers, archetype × league cell taxonomy locked 2026-05-18 pre-fit
- [PRE_REG_PROSPECT_PROJECTION_v1.md](D:/Draft%20Resolve/docs/PRE_REG_PROSPECT_PROJECTION_v1.md) through [v4](D:/Draft%20Resolve/docs/PRE_REG_PROSPECT_PROJECTION_v4.md) — each version locks cohort, features, models, calibration policy, validation metric before fitting
- [PRE_REGISTRATION_NBA_PLAYOFFS_25_26.md](D:/NBA%20Projections/PRE_REGISTRATION_NBA_PLAYOFFS_25_26.md) — 5 Tier-1 playoff hypotheses locked 2026-05-10
- All deviations from pre-reg are logged in published methodology docs alongside results

### 2. Falsification cascade

Each model passes through **multiple independent gates** before shipping:

- Naive event study → Propensity matching → Predictive validation (out-of-sample)
- LOYO cross-validation with **nested inner CV** for calibration + hyperparameter selection
- Decision rule: must beat baseline on pre-committed headline metric; null published with equal prominence to positive results

**Two pre-registered NULLs in regular-season paper**: Probe B contextual structure + PBP score-margin × possession cross-season. Published with same prominence as confirmations.

### 3. Cross-league replication

Same residual-class methodology validated across **NBA + WNBA + NCAA Division I Men's + NCAA Division I Women's**:

- **BLK × Center couples in 11/11 testable cells** across all 4 leagues × 2 projection methods (surgical + hierarchical NB2 Stan)
- **PTS × Center directional in 11/11** (formal significance reached at NCAA cohort sizes)
- **AST × deep-vet null replicates 6/6** across NBA cross-season + WNBA surgical 3-classifier matrix

### 4. LOO MSE deployment gate

A model's variance-class finding is only deployed to production if it clears a **1% population-MAE-improvement threshold** on leave-one-out. As of 2026-05-11, **0/11 cross-season-replicated mean offsets pass the gate** — methodological discipline preserves parsimony.

## Headline results — by product

### Prospect projection v4 (ensemble — newest)

- **Macro log loss 1.209** vs draft-pick-only baseline 1.448 (16.5% improvement)
- Beats baseline in **4 of 7 LOYO folds outright**, second-best in the other 3
- Per-fold std 0.081 — **lower than the baseline's 0.110** (more stable than the market)
- Ensemble weight `w=0.75 logistic + 0.25 calibrated ordinal` selected in **every single LOYO fold** — robust blend
- Macro F1 **0.395** vs baseline 0.092 (4.3× better)
- 2025 top: Cooper Flagg → star, E=5.53 seasons, P(star)=0.503

## Architecture — how the pieces compose

```
Pre-NBA features (Bart Torvik + Combine + Scout + G-League)
    │
    ├─→ Draft Resolve Prospect Projection v4 (ensemble)
    │       │
    │       └─→ Class Strength + Pick Valuation
    │
    └─→ NBA Projections cohort widening (rookie prior)
            │
            └─→ v4-lite Stan posterior (12-stat regular-season per-player)
                    │
                    ├─→ Team game prediction (Phase 2+3) → 65.31% Vegas-match
                    │       │
                    │       ├─→ Season Win Totals (aggregation)
                    │       └─→ Series Simulator (best-of-7 Monte Carlo)
                    │
                    ├─→ Playoff Residual Analysis (5 Tier-1 pre-registered hypotheses)
                    │
                    └─→ Playoff Player Projection v1 (regime-adjustment)

→ Wonka Resolve (firewall: audit-CSV only, no posterior co-design)
```

### Boundary architecture

- **Wonka ↔ NBA Projections** has a **firewall**: two Bayesian posteriors on the same truth → self-reference collapse risk. Audit-CSV contract is the only data flow.
- **Draft Resolve ↔ NBA Projections** has NO firewall: retrospective + pre-NBA→NBA translational, different epistemic domain, no collapse risk.

## What's not yet shipped (deferred, with reason)

### Playoff residual analysis v2 (Baseline A v6.1 re-scoring)
Pre-reg §2.5 requires both Baseline A (v6.1 conditional re-scoring) and Baseline B (cohort-mean) agree on disposition. v1 ships Baseline B preliminary results; Baseline A requires running v6.1 inference on playoff covariates — a heavier separate build. Deferred to playoff_residual_analyzer_v2.py.

### Playoff PBP score-margin × possession (Prediction 5)
Requires per-state clutch/garbage tagging across all 2025-26 playoff PBP. PBP data is ingested at [playoffs/round1/pbp_t0.parquet](D:/NBA%20Projections/data/parquet/playoffs/round1/pbp_t0.parquet) but the state-tagging pipeline runs as a separate step. Deferred.

### Playoff player projection v2 (Stan refit on playoff cohort)
v1 ships a population-level regime-ratio adjustment (median ratios applied to regular-season per-min rates). Full Stan refit on playoff-only cohort with proper hierarchical pooling is a multi-hour build, deferred.

### Per-team playoff-specific game model
v1 series simulator uses regular-season derived team strength. A playoff-conditioned game model (accounting for rotation tightening, opponent quality bracket filter, within-series adjustments) would tighten predictions. Deferred.

### Class Strength Index x Prospect Projection composition
Pick valuation v1 reports historical baselines + this-year v4 deltas side-by-side. A fully composed product (this-year EV of pick #N given this-year class strength × pick-range historical conditional distribution) is a v2 enhancement.

### 2025 universe expansion post-draft
v3/v4 prospect projection scores 79 combine-attendee 2025 prospects (deduplicated). After the actual draft, expanding the universe to all 60 drafted picks + post-draft signing/two-way contracts will let the model score the complete cohort and test calibration vs reality.
